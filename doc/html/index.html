KINOBOARDS version 1.0 release 6.5

Last Update: June 27, 1999

1. はじめに

きのぼーず(KINOBOARDS: Kinoboards Is Network Opened BOARD System)は， 以下を支援することを目的としています．

  • WWWの仕組みを用いて自由に意見交換を行なう．
  • そこで得られた知見をまとめ挙げ，後に役立てる．

つまり，一般的に「掲示板」と呼ばれている， 出会いとコミュニケーションが第一目的の書き捨て型のシステムとは， 目指しているところが異なります． そのためきのぼーずをチャットシステムの延長として捉えると， やや扱い辛いかもしれません． 逆に，記事がひたすら書き捨てられていくのに疲れてしまった方にとっては， 良い薬になる可能性があります．

2. 機能概要

きのぼーずの 現在のバージョン は，以下のような機能を持っています．

  • 書く
    • 新規にメッセージを書き込む機能
    • メッセージに対するリプライメッセージを書き込む機能
    • 投稿者の名前等の入力の手間を省くための，エイリアス機能
    • メッセージの内容に応じたアイコンを貼り付ける機能
    • リプライメッセージ書き込み時に，元メッセージを引用する機能
    • メッセージに自動的にハイパーリンクを埋め込む機能
    • jcode.plの機能を用いて，いわゆる半角仮名をいわゆる全角仮名へ変換する機能
    • 用意したファイル(書き溜めた日記等)を，メッセージとして取り扱う機能
  • 読む
    • タイトル一覧を表示する機能(リプライ順ソート，日付順ソート)
    • タイトル一覧を指定記事数毎にページ分割して表示する機能
    • メッセージ一覧を表示する機能(リプライ順ソート，日付順ソート)
    • あるメッセージへのリプライメッセージのみをまとめ読みする機能
    • メッセージを検索する機能
  • メイルとのリンク
    • リプライメッセージが書き込まれた時に， 元メッセージの作者に記事をメイルで自動配信する機能
    • 擬似メイリングリスト機能 （新規メッセージが書き込まれた時に，指定したユーザにメイルで自動配信できます）
    • コマンドラインから記事を書き込む機能 （.forwardや/etc/aliasesとの組み合わせで， 届いたメイルを新着記事として取り込めます．今のところリプライはできません）
  • 管理する
    • メッセージを削除する機能
    • メッセージを訂正する機能
    • 新規メッセージが書き込まれた時にメイルを自動配信するユーザを設定する機能
    • メッセージ表示順序を変更する機能
    • メッセージの元-リプライ関係を変更する機能

実際には，特に表示のカスタマイズに関して，もっと多くの機能を持っています． デモサイト や きのぼず設定ファイル で，その機能や設定の自由度を試してみてください．

3. 動作要件

きのぼーずの動作実績がある環境は， ENVIRONMENTドキュメント にまとめられています．

4. パッケージ

きのぼーずの配布パッケージに含まれているファイルについては， MANIFESTOドキュメント を参照してください． 最初の一度だけでいいですから， さらっと目を通しておいて頂けると嬉しいです．

5. インストール

インストール方法は，初めてのインストールか， それとも旧バージョンのバージョンアップかに応じて， 2種類にわかれています． 詳しくは INSTALLATIONドキュメント を参照してください．

6. インストール後のメンテナンス

インストール後の管理者の仕事は，以下のようなものです． 必ずしもやらなければならない，というわけではありませんが， 一応目を通しておくと，後で困らずに済むかもしれません．

  • 掲示板を新設する
  • 掲示板の機能を個別にカスタマイズする
  • 掲示板のアイコンを個別にカスタマイズする
  • 用意したファイルをメッセージ扱いする
  • 書き込みがあった時に，記事をメイルで自動配信する宛先を設定する
  • 記事を訂正する
  • 記事を削除する
  • ログを監視する

手順の詳細は MAINTENANCEドキュメント を参照してください．

8. Release Notes

R6.5では，以下の修正を行いました． 具体的な改版履歴は CHANGESファイル にまとめられています．

  • バグ修正
    • 森嶋さんの指摘で，スレッド表示において， 手動で削除した記事が表示されてしまうというバグを修正．
    • HORさんの指摘で，リプライメイル配信時にFROMヘッダが 元記事作者になってしまうというバグを修正．
    • ひ〜くんさんの指摘で， エラー画面から「タイトル一覧」へのリンクがおかしい（古い記事が表示される） というバグを修正．
    • 八村さんの指摘で，Post.plにおいてメイルのFromヘッダ解析に失敗する というバグを修正．
  • 機能追加
    • HORさんの指摘で，オペレーションログとエラーログを分離． ログファイルはlogディレクトリ内に置かれるように．
    • これまでみなさんに作って頂いたアイコンを，全てパッケージに入れた．
    • 諸星さんの指摘で，リプライ順での記事一括表示機能を追加．
    • ヘッダ部分にFORMによるジャンプメニューを追加．
  • 機能変更
    • 綾乃介さんのPatchを元に，ヘッダ部分のフォーマットを変更．
    • マキボンさんの指摘で，管理者用タイトル一覧と一般タイトル一覧を統合． 管理者用タイトル一覧から書き込みや削除を行なった時に， 一般タイトル一覧に戻ってしまわないように．
    • 綾乃介さんの指摘で， 記事の一覧表示の時にも掲示板ヘッダ（kb.board）を表示するようにした．
    • ひ〜くんさんの指摘で，index.htmlのHTMLヘッダ部分に charset宣言を追加．
    • 文香さんの指摘で，ログにREMOTE_USERを記録するように変更．
    • ネコ七さんの指摘で，新着記事の[new!]アイコンも文字列にできるように変更．
    • 自動配信メイルのSubjectからタグを削除するように変更．
    • 間宮さんの指摘で，記事の引用および訂正時に， 元記事中のタグを削ってしまわないように修正．
    • R_mitさんの指摘で，リプライまとめ読みおよび リプライ順記事一括表示画面において， fragment part（#〜）を使った記事内リンクを埋め込むように変更．
    • 間宮さんの指摘で，メイル投稿において， x-kb-commandの有無にかかわらず投稿できるように変更．
    • 諸星さんの指摘で，メイル投稿時， 記事中にx-kb-commandリテラルがあった場合にこれを自動的に削除するように変更．

不具合，変更点について指摘してくださったみなさん， どうもありがとうございました．m(..m

R6.5では上記機能修正の他に， パッケージ内容の変更（主にディレクトリ構成の変化）と， それに伴なって インストールマニュアル の大変更を行ないました． 前回インストールマニュアルを書き換えたのは2年くらい前かな． とある雑誌原稿のために書いたんですよね．

あの時点で，プロバイダ間の設定の違いを吸収するための 環境変数の利用，およびperlのcオプションによる構文チェックは， 他に見られない斬新なものだったと思ってますが（自画自賛するヤツ ^^;）， 今見直すとやっぱりもう，記述が古臭くなっちゃってましたねぇ．

あの頃の対象読者は 「比較的自由になる自分用サーバにインストールする人」と 「CGI利用可能なプロバイダにインストールする人」が半分ずつでした． 今や，後者のほうが多いでしょうねぇ．．． .htaccessを自力で設定する人なんて，もうほとんど居ないでしょう． CGIには恵まれた世の中になったもんです． というわけで今回は，おもいっきり後者の人向けに書き直しました． 新基軸はCGIWRAPおよびsuEXEC対策くらいかな? とはいえこれは，前回とは違い，ちょっと時代遅れ気味かな．

今後は．．．まずは（1年前! の）KB/1.2で実装した，認証機能を取り込みます． データファイルの互換性は保つつもりです．つまり，KB/1.0の延長で， たぶんR7.0としてリリースします． いつになるかな．．．本格的に暑くなる前には．．．とは，思っています． が，もはや誰も信じてくれないでしょうねぇ．^^;;
