KINOBOARDS version 1.0 release 6.5

Last Update: June 19, 1999

1. はじめに

きのぼーず(KINOBOARDS: Kinoboards Is Network Opened BOARD System)は， 以下を支援することを目的としています．

  • WWWの仕組みを用いて自由に意見交換を行なう．
  • そこで得られた知見をまとめ挙げ，後に役立てる．

つまり，一般的に「掲示板」と呼ばれている， 出会いとコミュニケーションが第一目的の書き捨て型のシステムとは， 目指しているところが異なります． そのためきのぼーずをチャットシステムの延長として捉えると， やや扱い辛いかもしれません． 逆に，記事がひたすら書き捨てられていくのに疲れてしまった方にとっては， 良い薬になる可能性があります．

2. 機能概要

きのぼーずの 現在のバージョン は，以下のような機能を持っています．

  • 書く
    • 新規にメッセージを書き込む機能
    • メッセージに対するリプライメッセージを書き込む機能
    • 投稿者の名前等の入力の手間を省くための，エイリアス機能
    • メッセージの内容に応じたアイコンを貼り付ける機能
    • リプライメッセージ書き込み時に，元メッセージを引用する機能
    • メッセージに自動的にハイパーリンクを埋め込む機能
    • jcode.plの機能を用いて，いわゆる半角仮名をいわゆる全角仮名へ変換する機能
    • 用意したファイル(書き溜めた日記等)を，メッセージとして取り扱う機能
  • 読む
    • タイトル一覧を表示する機能(リプライ順ソート，日付順ソート)
    • タイトル一覧を指定記事数毎にページ分割して表示する機能
    • メッセージ一覧を表示する機能(リプライ順ソート，日付順ソート)
    • あるメッセージへのリプライメッセージのみをまとめ読みする機能
    • メッセージを検索する機能
  • メイルとのリンク
    • リプライメッセージが書き込まれた時に， 元メッセージの作者に記事をメイルで自動配信する機能
    • 擬似メイリングリスト機能 （新規メッセージが書き込まれた時に，指定したユーザにメイルで自動配信できます）
    • コマンドラインから記事を書き込む機能 （.forwardや/etc/aliasesとの組み合わせで， 届いたメイルを新着記事として取り込めます．今のところリプライはできません）
  • 管理する
    • メッセージを削除する機能
    • メッセージを訂正する機能
    • 新規メッセージが書き込まれた時にメイルを自動配信するユーザを設定する機能
    • メッセージ表示順序を変更する機能
    • メッセージの元-リプライ関係を変更する機能

実際にはもっと多くの機能を持っています． よその掲示板が持っている機能のほとんどは， 設定を変更するだけで簡単に実現できるでしょう． デモサイト や きのぼず設定ファイル で，その多彩な機能と，設定の自由度を確認してください．

6. インストール後のメンテナンス

インストール後の管理者の仕事は，以下のようなものです． 必ずしもやらなければならない，というわけではありませんが， 一応目を通しておくと，後で困らずに済むかもしれません．

  • 掲示板を新設する
  • 掲示板の機能を個別にカスタマイズする
  • 掲示板のアイコンを個別にカスタマイズする
  • 用意したファイルをメッセージ扱いする
  • 書き込みがあった時に，記事をメイルで自動配信する宛先を設定する
  • 記事を訂正する
  • 記事を削除する
  • ログを監視する

手順の詳細は，MAINTENANCEファイルを参照してください．

8. Release Notes

R6.4では，以下の修正を行いました． 今回の修正点は，ここ数回のリリースの修正点の数と比べると，やや少なめかな?

  • 記事訂正後に「リプライ元の記事へ」と表示されるバグを修正． 「訂正した記事へ」に．
  • URL中の`&'を`&amp;'にencodeするように修正．
  • 自動配信メイルのFrom:ヘッダに，記事投稿者を入れるように変更． 従来は掲示板管理者が入っていた．
  • R_mitさんの指摘で，PATH_INFO無条件信頼方式を廃止． kb.cgiの先頭で，kbディレクトリのパスを指定しなければ動かないように変更．
  • NOBZILLAさんの指摘で，投稿日付による検索機能を追加．
  • 大垣さんの指摘で，環境変数REMOTE_HOSTが空の時， 替わりにREMOTE_ADDRを使うように変更．
  • Sageさんの指摘で，記事中のタグにstyle，lang等のattributeが指定できない， というバグを修正．
  • 記事中のタグを解析して安全なタグのみを残すルーチンのバグを修正． ここのところリリースのたびに修正&エンバグしてる．．．
  • 諸星さんの指摘で，HEADリクエストで起動された際， Last-Modifiedヘッダを返すように変更． ただし掲示板指定がある場合（kb.cgi?b=test等）のみ． 具体的には「HEAD /〜/kb.cgi?b=test HTTP/1.0」のように リクエストする必要がある．
  • 岩渕さん他の指摘により，コマンドラインから投稿機能を追加． 「kb.cgi 掲示板名 POST」というコマンドを起動し，記事を標準入力から流し込む． ただし記事中に「x-kb-command: POST」という文字列が含まれていなければならない．
  • 混乱を避けるため，利用可能なタグを指定を，cgi.plから取り除いた． 利用可能タグはkb.cgiのみで指定する．
  • 大垣さんの指摘で， kbディレクトリがウェブサイトのトップディレクトリに等しい場合に， $CGIDIR_NAMEを取り損ねる不具合を修正．
  • 2000年対応．．．これで終わり．．．と思う．

不具合，変更点について指摘してくださったみなさん， どうもありがとうございました．m(..m

「コマンドラインから投稿」の機能は， 手動投稿や他スクリプトからの投稿に使えますが， 想定している主な利用方法は，メイルを自動的に記事として取り込む， というものです．ただし現在は，すべて新規記事として取り込まれます． スレッドを継ぐには，管理者がいちいち手でやらなければなりません． 具体的な利用方法がわからない方は， きのぼずML までどうぞ．

毎度「次こそ1.2」とか書いてますが， ML統合も面白そうだし．．．どうしよう．．．
