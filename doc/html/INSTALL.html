installation

Last Update: 9/18/96

インストール方法は，旧バージョン(R2.3)をインストール済みか， それとも初めてのインストールなのか， また，あなたが利用するhttpサーバのCGIの稼働設定に応じ， 以下の3種類にわかれています．

  • 1. R2.3をインストール済み
  • 初めてのインストール
    • 2. CGIを自分のディレクトリ内に持てる場合
    • 3. CGIは共通のディレクトリに置かなければならない場合

1. R2.3からのバージョンアップ

release 2.3から変更のあったファイルで， 今回のpatchファイル(kb10_24.patch)によってバージョンアップされるのは， 以下のファイル群です．

  • kb.cgi
  • kb.ph
  • docディレクトリ以下のドキュメント
  • cgi.pl
  • tag_secure.pl
  1. patchファイル(kb10_24.patch)をダウンロードしてください．
  2. release 2.3のkbディレクトリへ行き，システムを止め， 念のためバックアップを取ってください． もし記事が消えたり，設定が変わって動かなくなっても，責任は取れません(^_^;． バックアップは必ず取っておいてくださいね．
    cd ~/public_html/kb
    touch .lock.kb
    cd ..
    tar cvfp kb.tar kb
  3. kbディレクトリに対して，patchを当ててください．
    cd ~/public_html/kb
    cd ..
    patch < kb10_24.patch
  4. システムのロックを外し，システムが正しく動作することを確認してください．
    rm .lock.kb

2. ユーザがCGIを自分のディレクトリ内に持てる設定

以下の手順でインストールを行ないます．

  1. WWWとしてアクセス可能なディレクトリで， 配布パッケージ(tar + gzip)を展開します．
    gzip -dc kb10_24.tar.gz | tar xvfp -
  2. 注意!: 展開時，tarのpオプションを忘れないようにしてください． 本システムはパーミッションにシビアです! m(..)m
  3. kbというディレクトリが作られ，その中にプログラム本体が展開されます．
  4. サーバの設定によっては，そのディレクトリでCGIプログラムが動くために， .htaccessというファイルの設定が必要です． 設定方法等は，サーバ管理者の方に問い合わせてください．
  5. 展開してできたkbディレクトリに，jcode.plを置きます． SRAのftp site から取ってきて，他人でも読めるようにして置いてください． jcode.plは(株)SRAの歌代さんが作られたフリーウェアで， これがなければ日本語のコード変換ができません．
    chmod a+r jcode.pl
  6. kb.cgiの先頭行のperlのpathを，動作環境に合わせて設定します．
    #!/usr/local/bin/perl → #!/usr/local/bin/GNU/perl5
  7. もう動くはずなので，適当なブラウザで， 配布パッケージを展開した時に作られたkbディレクトリにある， index.htmlにアクセスしてみてください．
  8. とりあえず動くことを確認したら，kb.phを書き換えて， きのぼーずをあなたのサイト用に設定し直してください． ただし，特に変更が必要なのは「管理者のメールアドレス」のみだと思います (必ず変更してくださいね ^_^;)． 「システム設定」や「メッセージ」は，お好みで変更してください．

3. CGIを共通のディレクトリに置く設定

この場合，CGI(kb.cgi)と，その他のファイルを置くディレクトリを， 分けなければなりません． 以下の手順でインストールを行ないます．

  1. WWWとしてアクセス可能なディレクトリで， 配布パッケージ(tar + gzip)を展開します．
    gzip -dc kb10_24.tar.gz | tar xvfp -
  2. 注意!: 展開時，tarのpオプションを忘れないようにしてください． 本システムはパーミッションにシビアです! m(..)m
  3. kbというディレクトリが作られ，その中にプログラム本体が展開されます．
  4. 展開してできたkbディレクトリに，jcode.plを置きます． SRAのftp site から取ってきて，他人でも読めるようにして置いてください． jcode.plは(株)SRAの歌代さんが作られたフリーウェアで， これがなければ日本語のコード変換ができません．
    chmod a+r jcode.pl
  5. kb.cgiの先頭行のperlのpathを，動作環境に合わせて設定します．
    #!/usr/local/bin/perl → #!/usr/local/bin/GNU/perl5
  6. kb.cgiを，CGIを置く共通のディレクトリに移します．
    (共通のディレクトリが/usr/local/etc/htdocs/cgi-bin/の場合)
    mv kb.cgi /usr/local/etc/htdocs/cgi-bin/
  7. index.htmlを書き換えます． index.htmlを見ると，「テスト」掲示板へのリンクが，
    <a href="kb.cgi?b=test&c=v&num=0">テスト</a>
    となっていると思います．これを次のように変更してください．
    <a href="(A)/kb.cgi/(B)?b=test&c=v&num=0">テスト</a>
    • (A): kb.cgiを置いたディレクトリのURL(例えば/cgi-bin)
    • (B): 配布パッケージを展開した時に作られたkbディレクトリのURL(例えば/~nakahiro/kb)
    すなわち，「kb.cgi」→「(A)/kb.cgi/(B)」と変更します．上の例なら，
    <a href="/cgi-bin/kb.cgi/~nakahiro/kb?b=test&c=v&num=0">テスト</a>
    になります(上の例でのサンプルが，index.html.cgi-binとして用意してありますので， 参考にしてください))．
  8. 同様にして，test/.boardに書かれているkb.cgiへのリンクを， 全て上のように(「kb.cgi」→「(A)/kb.cgi/(B)」のように)書き換えてください． index.htmlと同様に.board.cgi-binとして用意してあります．参考にしてください．
  9. これで動くはずなので，適当なブラウザで， 配布パッケージを展開した時に作られたkbディレクトリにある， index.htmlにアクセスしてみてください．
  10. とりあえず動くことを確認したら，kb.phを書き換えて， きのぼーずをあなたのサイト用に設定し直してください． ただし，特に変更が必要なのは「管理者のメールアドレス」のみだと思います (必ず変更してくださいね ^_^;)． 「システム設定」や「メッセージ」は，お好みで変更してください．
